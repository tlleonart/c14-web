# Quickstart: Landing Page para Servicios de IA

**Date**: 2026-01-28
**Feature**: 001-landing-page

## Prerequisites

- Node.js 18+ installed
- npm or pnpm package manager
- Git
- A Convex account (free at convex.dev)
- A Resend account for emails (free at resend.com)

## Quick Setup (5 minutes)

### 1. Clone and Install

```bash
git clone <repository-url>
cd c14_v2
git checkout 001-landing-page
npm install
```

### 2. Initialize Convex

```bash
npx convex dev
```

This will:
- Prompt you to log in with GitHub
- Create a Convex project
- Generate `.env.local` with `CONVEX_DEPLOYMENT` and `NEXT_PUBLIC_CONVEX_URL`

Keep this terminal running during development.

### 3. Configure Environment

Create/update `.env.local`:

```bash
# Convex (auto-generated by npx convex dev)
CONVEX_DEPLOYMENT=dev:your-deployment-name
NEXT_PUBLIC_CONVEX_URL=https://your-deployment.convex.cloud

# Email (get from resend.com dashboard)
RESEND_API_KEY=re_xxxxxxxxxxxx
CONTACT_EMAIL=your-email@example.com
```

### 4. Run Development Server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000)

## Common Tasks

### Add a New Service

Edit `src/modules/landing/data/services.ts`:

```typescript
export const services: Service[] = [
  // ... existing services
  {
    id: 'new-service',
    name: 'Nuevo Servicio',
    description: 'Descripción del servicio...',
    icon: 'icon_name', // Material Symbols icon
    features: ['Feature 1', 'Feature 2']
  }
]
```

### Modify Contact Form Fields

1. Update Zod schema in `src/server/api/routers/contact.ts`
2. Update Convex schema in `convex/schema.ts`
3. Update ContactForm component in `src/modules/contact/components/ContactForm/`

### Change Design Tokens

Edit `src/shared/styles/variables.css`:

```css
:root {
  --color-primary: #your-new-color;
  /* ... */
}
```

### Test Contact Form Locally

1. Submit form at `http://localhost:3000`
2. Check Convex dashboard for new record
3. Check email inbox (if Resend configured)

### View Contact Submissions

Go to your Convex dashboard:
1. Open [dashboard.convex.dev](https://dashboard.convex.dev)
2. Select your project
3. Navigate to "Data" → "contactRequests"

## Deployment

### Deploy to Vercel

```bash
npm run build  # Verify build succeeds

# Then deploy via Vercel CLI or GitHub integration
```

### Environment Variables for Production

Set these in your deployment platform:

| Variable | Description |
|----------|-------------|
| `CONVEX_DEPLOYMENT` | Production deployment name |
| `NEXT_PUBLIC_CONVEX_URL` | Production Convex URL |
| `RESEND_API_KEY` | Resend API key |
| `CONTACT_EMAIL` | Email to receive contacts |

### Deploy Convex to Production

```bash
npx convex deploy
```

## Troubleshooting

### "Module not found" errors

```bash
rm -rf node_modules .next
npm install
npm run dev
```

### Convex connection issues

1. Ensure `npx convex dev` is running
2. Check `.env.local` has correct URLs
3. Restart dev server

### Form submission not working

1. Check browser console for errors
2. Verify Convex mutations are deployed
3. Check tRPC API route exists at `/api/trpc`

### Emails not sending

1. Verify `RESEND_API_KEY` is set
2. Check Resend dashboard for errors
3. Verify sender domain is configured
