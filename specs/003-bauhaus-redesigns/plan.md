Create three alternative landing page designs at `/v2`, `/v3`, `/v4` using CSS Variable Overrides + Section Wrapper approach. Each variant page reuses all existing React components unchanged, wrapping them in styled containers that override CSS custom properties and apply structural CSS changes via element selector specificity. ## Architecture: CSS Variable Overrides + Section Wrappers

### How It Works

1. **CSS Custom Property Cascade**: All component CSS modules reference CSS variables (`--color-*`, `--spacing-*`, `--font-*`, `--radius-*`, etc.). CSS custom properties cascade through the DOM regardless of CSS Module class scoping. Overriding these on a parent element re-themes all children.

2. **Section Wrapper Elements**: Each variant page wraps original components in styled `<div>` elements:
   ```tsx
   <div className={styles.heroWrap}><Hero /></div>
   ```
   Wrapper selectors like `.heroWrap h1` have specificity (0,1,1) which overrides component CSS Module classes (0,1,0), enabling typography, layout, and decorative changes.

3. **Pseudo-elements for Decoration**: Wrapper divs provide anchor points for `::before`/`::after` pseudo-elements — enabling V2's geometric shapes and section separators without modifying component DOM.

### Hardcoded Values Requiring Wrapper Overrides

A few component CSS values are hardcoded (not CSS variables):

| Component | Hardcoded Value | Override Strategy |
|-----------|-----------------|-------------------|
| Header `.header` | `rgba(10, 10, 10, 0.8)` background | `.headerWrap header { background: ... }` |
| Hero `.glow` | Primary color rgba gradient | `.heroWrap > section div[aria-hidden]` or hide via `display: none` |
| Hero `.grid` | White rgba grid lines | Same wrapper override approach |
| Services `.card:hover` | Primary color rgba | `.servicesWrap article:hover { background: ... }` |
| Services `.cardIcon` | Primary color rgba | `.servicesWrap article > div:first-child { background: ... }` |

### What Does NOT Need Overriding

- Colors via `--color-*` variables: cascade automatically
- Typography via `--font-*`, `--text-*`: cascade automatically
- Spacing via `--spacing-*`: cascade automatically
- Border radius via `--radius-*`: cascade automatically
- Shadows via `--shadow-*`: cascade automatically
- Transitions via `--transition-*`: cascade automatically
- Borders via `--color-border*`: cascade automatically
- Glass effects via `--glass-*`: cascade automatically

--glass-background: transparent
--glass-blur: 0
```

**Wrapper CSS additions:**
- All h1, h2, h3: `text-transform: uppercase; letter-spacing: 0.12em; font-weight: 900`

- Geometric decorative shapes via `::before`/`::after` on wrappers (circles, triangles using clip-path)
- Service cards: colored left border accent (alternating blue/yellow/primary)
- Hero: replace glow with geometric shape composition
- Grid layout adjustments via wrapper selectors where needed

```

**Wrapper CSS additions:**
- Extreme typography: hero h1 at `clamp(4rem, 12vw, 10rem)`, sections h2 at `clamp(3rem, 8vw, 7rem)`

- Remove all decorative effects (glow, grid, glass) — hide `[aria-hidden]` elements
- Full-bleed sections: remove max-width constraints on some sections
- Overlapping sections via negative margins
- Rotated accent text via `::before` with `transform: rotate(-90deg)` on section wrappers
- CTA and interactive elements use `--color-primary` background — only color in the page
